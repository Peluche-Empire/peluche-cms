## Project Overview

Multi-tenant Payload CMS v3 running on Cloudflare Workers with D1 (SQLite) database and R2 storage for media. Built on Next.js 15 with the OpenNext Cloudflare adapter. Requires a **paid** Cloudflare Workers plan due to bundle size limits (3MB free tier).

This CMS serves multiple sites under the **Pallucci Empire** umbrella, replacing DatoCMS:
- **Bassyana** — English agriculture/bioproducts blog
- **Goonie Nomada** — Spanish travel/digital nomad blog

Deployed at: `https://peluche-cms.peluche-empire.workers.dev`

## Commands

```bash
# Development
pnpm dev                      # Start local dev server (Wrangler auto-binds D1/R2)
pnpm devsafe                  # Clean start (removes .next and .open-next first)
pnpm wrangler login           # Authenticate with Cloudflare (required before first dev)

# Build & Deploy
pnpm run deploy               # Full deploy: runs migrations, builds, deploys to Cloudflare
pnpm run deploy:database      # Run migrations on remote D1 only
pnpm run deploy:app           # Build and deploy Worker only
pnpm preview                  # Local preview of production build

# Type Generation (run after schema changes)
pnpm generate:types           # Generates both Cloudflare and Payload types
pnpm generate:importmap       # Regenerate import map after adding/modifying components

# Code Quality
pnpm lint                     # ESLint (next/core-web-vitals + next/typescript)
npx tsc --noEmit              # TypeScript type checking

# Testing
pnpm test:int                 # Integration tests (Vitest + jsdom)
pnpm test:e2e                 # E2E tests (Playwright, spins up dev server)

# Database
pnpm payload migrate:create   # Create a new migration after schema changes
```

## Architecture

**Cloudflare-specific adapter chain:** `Next.js -> OpenNext (@opennextjs/cloudflare) -> Cloudflare Worker`

- `wrangler.jsonc` — Cloudflare bindings config: D1 database (`peluche-cms-db`), R2 bucket (`peluche-cms-media`), compatibility flags
- `open-next.config.ts` — OpenNext Cloudflare adapter config
- `next.config.ts` — Wrapped with `withPayload()`, marks `jose` and `pg-cloudflare` as external for Workers compatibility
- `src/payload.config.ts` — Main Payload config. Uses `sqliteD1Adapter` for DB, `r2Storage` for media uploads, and `multiTenantPlugin` for tenant scoping. Includes a custom console-based logger for Workers (production only, since `pino-pretty` uses unsupported Node APIs)

**Route groups:**
- `src/app/(frontend)/` — Public-facing Next.js pages
- `src/app/(payload)/` — Payload admin panel and API routes (auto-generated, don't modify)
- `src/app/my-route/` — Example custom API route

**Multi-tenancy:** Uses `@payloadcms/plugin-multi-tenant`. The plugin auto-creates a `tenants` array field on Users and adds a `tenant` relationship field to all scoped collections. Tenant selector in admin UI filters content per-tenant.

**Collections:** `src/collections/`

| Collection | Slug | Tenant-scoped | Notes |
|---|---|---|---|
| Users | `users` | via tenants array | Has `roles` field: `super-admin` (all tenants) / `editor` |
| Media | `media` | Yes | Uploads to R2, no crop/focalPoint (no sharp on Workers) |
| Tenants | `tenants` | N/A | `name` + `slug` |
| Articles | `articles` | Yes | Rich text (Lexical), heroImage, category, country (conditional), tags |
| Categories | `categories` | Yes | `title`, `slug`, `description` |
| Tags | `tags` | Yes | `tagName` |
| Countries | `countries` | Yes | Goonie Nomada-specific; `countryName`, `slug` |
| Tools | `tools` | Yes | Goonie Nomada-specific; title, image, category, url, ctaText |
| ToolCategories | `tool-categories` | Yes | Goonie Nomada-specific; `title` |

The `country` field on Articles uses an admin `condition` to hide when the tenant is Bassyana.

**Migrations:** `src/migrations/` — D1 SQLite migrations generated by Payload

**Path aliases:** `@/*` maps to `./src/*`, `@payload-config` maps to `./src/payload.config.ts`

## Cloudflare Workers Constraints

- No `sharp` — image crop and focalPoint are disabled in the Media collection
- No `pino-pretty` — production uses a custom console-based JSON logger
- GraphQL has known upstream issues in Workers runtime
- Bundle size limit of 3MB on free plan; paid plan required
- Media collection uses `skipSafeFetch: true` to avoid `undici` diagnostic channel errors (safe because Workers block private IP ranges)
- `PAYLOAD_SECRET` must be set as a Cloudflare secret (generate with `openssl rand -hex 32`) and in `.env` for local dev
- `.dev.vars` and `.env` are gitignored and contain `PAYLOAD_SECRET` for local development

## Payload CMS Patterns

Key rules from AGENTS.md (`.cursor/rules/` has detailed reference files):

- **Always pass `req`** to nested Payload operations in hooks (transaction safety)
- **Always set `overrideAccess: false`** when passing `user` to Local API calls
- **Use context flags** to prevent infinite hook loops
- **Components use file paths**, not direct imports — regenerate import map after changes
- **Run `pnpm generate:types`** after any collection/field schema changes
- Generated types live in `src/payload-types.ts`
